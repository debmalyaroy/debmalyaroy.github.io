About me

I build systems that think, adapt, and act — not just systems that respond.

With 18+ years architecting large-scale distributed backends at Coupang, Grab, and Zeta, I'm now fully focused on the frontier: Agentic AI systems — autonomous agents that plan, use tools, remember context, and orchestrate other agents to solve problems no single model can tackle alone.

My north star: distributed systems expertise × LLM orchestration = production-grade agentic infrastructure. I write, build, and share everything I learn about this shift.

What I'm building

▸ agent-router
Intent-Based Agent Router
Replacing static API gateways with LLM-powered routing that understands user intent and dynamically dispatches to specialist agents.
▸ multi-agent-orchestrator
Multi-Agent Orchestration Framework
Planner → Retriever → Executor → Validator pipeline with shared memory, tool registries, and feedback loops at scale.
▸ agent-memory
Persistent Agent Memory Layer
Working, episodic, semantic, and procedural memory for agents — built on vector stores, graph DBs, and skill libraries.
▸ self-healing-infra
Self-Healing Distributed Systems
Autonomous monitoring agents that detect anomalies, reroute traffic, restart pods, and learn from incidents — zero human paging.

Architecture philosophy

→ API Gateway → Agent Router — Static routing is dead. Intent-aware dispatch is the new gateway pattern.
→ Stateless REST → Stateful Agents — Context that persists across sessions is the foundation of useful autonomous systems.
→ Single Agent → Swarms — Specialised agents running in parallel outperform monolithic models every time.
→ On-call pages → Self-healing — A system that wakes up a human is a system that hasn't finished being built.
